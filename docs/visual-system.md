# 《别点错》视觉系统规范 v1.0

> **核心定位**：极简骨架 + 高饱和点缀 + 手绘表情灵魂。
> 用最低美术成本做出"看一眼就想截图发朋友"的效果。

---

## 1. 视觉关键词

```
主关键词        辅助关键词
──────────     ──────────────
极简            白底、大留白、零装饰
反差            安静背景 × 夸张表情
荒诞            不正经、不对劲、哪里怪怪的
轻手绘          2px 黑线描边、微微不工整
梗图感          表情包级别的情绪表达
高饱和点缀      整体灰白，关键元素一眼看到
```

**一句话总结**：一个看起来极其正常的界面，里面住着一群不正常的东西。

## 2. 色彩系统

### 2.1 色板

```
┌─────────────────────────────────────────────┐
│                  色彩系统                      │
├──────────┬──────────┬──────────────────────────┤
│  角色     │  色值     │  用途                    │
├──────────┼──────────┼──────────────────────────┤
│  背景白   │ #FAFAFA  │ 全局页面背景              │
│  纸张白   │ #FFFFFF  │ 卡片、弹层背景            │
│  主文字   │ #1A1A2E  │ 标题、正文                │
│  次文字   │ #6B7280  │ 说明文字、提示            │
│  淡灰     │ #E5E7EB  │ 分割线、禁用态            │
│  ────────│──────────│────────────────────────── │
│  通关绿   │ #22C55E  │ 通关按钮、成功状态        │
│  失败红   │ #EF4444  │ 失败状态、警告            │
│  主题蓝   │ #3B82F6  │ 主要交互按钮              │
│  陷阱黄   │ #F59E0B  │ 误导性元素、提示高亮      │
│  酱紫     │ #8B5CF6  │ 特殊/成就/稀有            │
│  ────────│──────────│────────────────────────── │
│  遮罩黑   │ #000/60% │ 弹层背景遮罩              │
│  阴影     │ #000/8%  │ 卡片投影                  │
└──────────┴──────────┴──────────────────────────┘
```

### 2.2 使用原则

```
1. 背景永远是 #FAFAFA 或 #FFFFFF —— 安静
2. 一个画面只允许一种高饱和色作为"视觉焦点"
3. 关卡内的误导元素用陷阱黄 #F59E0B 强化吸引力
4. 真正的通关元素反而低调（灰色、小字、角落）
   → 视觉设计本身服务于"反套路"玩法
```

## 3. 字体与字号

### 3.1 字体栈

```css
/* 正文：系统字体优先，零网络开销 */
--font-body: -apple-system, BlinkMacSystemFont, "Segoe UI",
             "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei",
             sans-serif;

/* 手绘风：用于关卡标题、搞笑文案（一个字体足够） */
/* 推荐：站酷快乐体（免费商用）或 ZCOOL KuaiLe（Google Fonts） */
--font-fun: "ZCOOL KuaiLe", "Comic Sans MS", cursive;
```

### 3.2 字号体系（基于 rem，根 16px）

| Token | 尺寸 | 行高 | 用途 |
|-------|------|------|------|
| `--text-xs` | 12px / 0.75rem | 1.4 | 辅助说明、版本号 |
| `--text-sm` | 14px / 0.875rem | 1.5 | 次要文字、提示 |
| `--text-base` | 16px / 1rem | 1.5 | 正文默认 |
| `--text-lg` | 20px / 1.25rem | 1.4 | 按钮文字、弹层正文 |
| `--text-xl` | 24px / 1.5rem | 1.3 | 页面标题 |
| `--text-2xl` | 32px / 2rem | 1.2 | 关卡序号、大标题 |
| `--text-3xl` | 48px / 3rem | 1.1 | 首页 LOGO、数字强调 |
| `--text-hero` | 72px / 4.5rem | 1.0 | 倒计时数字、戏剧性场景 |

### 3.3 使用规则

```
1. 系统字体用于：页面框架、按钮、提示、设置
2. 手绘字体仅用于：
   - 关卡标题（"通关"/"不要点"）
   - 搞笑评语（"你也太听话了吧"）
   - 首页 LOGO
3. 同一画面不超过 2 种字体
4. 关卡内游戏对象的文字由 Phaser 渲染（用 BitmapFont 保性能）
5. Vue DOM 层的弹层/HUD 用系统字体
```

### 4.4 关卡内按钮 vs 应用壳按钮

```
┌─────────────────────────────────────────┐
│  应用壳按钮（Vue DOM）                    │
│  - 标准圆角矩形                          │
│  - 系统字体                              │
│  - 统一样式                              │
│  ─────────────────────────────────────── │
│  关卡内按钮（Phaser Canvas）              │
│  - 可以任何形状、大小、位置              │
│  - 手绘字体                              │
│  - 每关不同，服务于关卡设计              │
│  - 用 Phaser NineSlice / Graphics 绘制  │
└─────────────────────────────────────────┘

铁律：两种按钮的样式不要统一。
应用壳按钮越"正常"，关卡内按钮的"不正常"就越有反差。
```

## 6. 反馈动效

### 6.1 动效清单

| 名称 | 触发场景 | 实现方式 | 参数 |
|------|---------|---------|------|
| `btn-bounce` | 按钮点击 | CSS transform | scale 1→0.92→1, 150ms, ease-out |
| `card-spring` | 弹层出现 | CSS transform | scale 0.85→1.03→1, 350ms, spring |
| `slide-up` | 弹层/toast 入场 | CSS transform | translateY(20px)→0, 300ms |
| `shake` | 失败/错误操作 | CSS animation | translateX ±4px, 3 次, 200ms |
| `confetti` | 通关 | Canvas（Phaser 粒子） | 50 个彩色方块向上喷射, 1.5s |
| `fade-zoom` | 关卡切换 | CSS transition | opacity + scale, 250ms |
| `float-up` | 通关文字 | Phaser tween | y -30px + alpha 0, 800ms |
| `pulse` | 提示高亮 | CSS animation | scale 1→1.08→1, 1s, infinite |
| `wobble` | 误导元素 | CSS animation | rotate ±3°, 2s, infinite |

### 6.2 Phaser 内动效 vs Vue DOM 动效

```
Phaser Canvas 层（game-core 管）：
├── confetti 粒子喷射
├── 游戏对象的 shake / wobble / float
├── 对象出现/消失的 scale + alpha tween
└── 拖拽释放的弹性回弹

Vue DOM 层（web-app 管）：
├── 弹层 card-spring / slide-up
├── 按钮 btn-bounce
├── 页面转场 fade-zoom
├── 提示 toast slide-up / slide-down
└── 关卡编号翻转动画
```

### 6.3 动效三原则

```
1. 快：所有动效 ≤ 400ms（通关撒花例外）
2. 弹：用 spring / ease-out，不用 linear
3. 少：一个时刻最多同时播 2 个动效，否则眼花
```

## 8. 图标与表情素材风格

### 8.1 表情系统

这是整个视觉体系的灵魂。用极少量的简笔线条表情传递最大情绪。

```
通关表情池（随机选用）：
  ( ͡° ͜ʖ ͡°)    得意
  (≧▽≦)       开心
  ᕕ( ᐛ )ᕗ     嘚瑟
  ( •̀ᄇ• ́)ﻭ    比耶
  ╰(*°▽°*)╯    欢呼

失败表情池：
  (╥_╥)        委屈
  (ᗒᗣᗕ)        无语
  ( ˘•ω•˘ )    呆滞
  (╯°□°)╯     崩溃
  ┐(´д`)┌      摊手

特殊状态：
  ( ˘ω˘ )zZ    加载中
  (°ロ°)!      惊讶（假失败后）
  ಠ_ಠ          审视（提示前）
```

### 8.2 表情渲染方式

```
方案：直接用文本渲染（Unicode 颜文字）
优势：
  - 零美术成本
  - 天然跨平台
  - 大字号就是大表情
  - 方便随时新增

备选方案（v0.2+）：
  - SVG 手绘表情（更有个性）
  - Lottie 动画表情（更有动感）
```

### 8.3 图标

```
来源：Lucide Icons（MIT 许可，线条风格，与极简主调一致）
尺寸：24×24px（标准）/ 20×20（小）/ 32×32（强调）
颜色：单色，跟随文字色
使用：
  ⚙  设置
  ←  返回
  ⏸  暂停
  🔊  音效开
  🔇  音效关
  🔒  锁定
  ✅  已通关
  ▶  当前关
  💡  提示
  ✕  关闭
```

### 8.4 关卡内素材风格

```
风格：2px 黑色描边 + 单色或渐变填充
工具：Figma / Excalidraw / AI 生成后手动调整

关卡素材分类：
├── 按钮类     → 圆角矩形，纯色填充
├── 角色类     → 简笔火柴人 / 圆形头+表情
├── 物件类     → 太阳、门、灯泡、手机 → 极简图标风
├── UI 仿真类  → 假广告、假 loading、假弹窗 → 仿真但略歪
└── 文字类     → 手绘字体渲染

关键：
  关卡素材要"微微不工整"——线条有一点点歪
  这种不工整传递"不正经"的调性
  但不能真的丑，要"可爱的歪"
```

```

  ui-      → UI 仿真元素

  -on/-off       → 开关状态

```

## 11. 设计总结

```
  这套视觉系统的核心矛盾就是产品的核心体验：

  ┌─────────────────────────────────────────┐
  │                                           │
  │   正常的壳    ×    不正常的内容             │
  │                                           │
  │   极简的界面  ×    夸张的表情               │
  │                                           │
  │   标准的按钮  ×    骗人的按钮               │
  │                                           │
  │   安静的背景  ×    离谱的反转               │
  │                                           │
  └─────────────────────────────────────────┘

  应用壳越正经，关卡内容越荒诞，反差感越强。
  视觉设计不是装饰，而是游戏玩法的同谋。
```
